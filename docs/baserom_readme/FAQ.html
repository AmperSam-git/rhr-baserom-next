Frequently Asked Questions

"What is Callisto?"

It's a build tool that manages your project and does all the usual romhacking things through a simple user interface with a few actions. Callisto designed to run tools only when needed and avoid potential conflicts between resources, this way it insures your hack is cleanly constructed whenever you make changes. The big upside of using this is if something goes wrong then you can easily pinpoint what that was and remove or fix it so the next time you build it is not included in your project.

See the Callisto readmes in the docs folder for more details information about how to use this tool.

"What happened to the build scripts?"

In an effort to keep the baserom lean, we're going all-in on the Callisto build tool. The build batch files, while useful, were also a bit of maintainance burden and didn't work for some people.

"How do I remove things from the baserom?"

Callisto makes it easy to remove items from the baserom without a problem as it re-builds your ROM from scratch avoiding resource conflicts. Removing resources, like blocks or sprites, is as straightforward as editing the list of the respective tool, or removing things from the "resources.toml" in the buildtool folder and running the Update action. Afterwards you can then delete those resources, if desired.

"How do I hide the status bar?"

This baserom comes with several objects to toggle features in levels, including the status bar. Check out the UberASM Objects documentation for details.

"Does this have Fast Retry?"

Yes! By default the baserom is configured to be "vanilla" but you can turn on the retry in a level using a custom object. The UberASM Objects has a section for the retry system with details. Otherwise you can make hack-wide changes to the retry system, by editing its config files in tools/uberasmtool/retry_config.

"Why are my block graphics not looking right?"

Many of the custom global block in the baserom use ExGraphics, loading E17 into the FG2 slot in the Super GFX Bypass dialog will make the blocks appear as they are intended to in other levels.

"Is the baserom SA-1 compatible?"

Yes! The baserom's resources are all compatible with SA-1, but the baserom itself is configured to be FastROM by default. However to use SA-1 it only takes a couple steps to switch over to it:

  1. In the "project.toml" file in the buildtool folder, change the initial_patch setting to use the SA-1 option.
  2. Rebuild your project in Callisto to make the change.
