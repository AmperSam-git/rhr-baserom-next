Frequently Asked Questions

"What is Callisto?"

It's a build system that makes it easier to build the rom from scratch, by doing this it avoids potential conflicts between resources and insures your hack is cleanly constructed whenever you make changes. The big upside of using a build tool is if something goes wrong then you can easily pinpoint what that was and remove it so the next time you build it is not included in your project.

"How do I remove things from the baserom?"

Callisto makes it easy to remove items from the baserom without a problem as it re-builds your ROM from scratch avoiding resource conflicts. Removing resources, like blocks or sprites, is as straightforward as editing the list of the respective tool, or removing things from the "resources.toml" in the buildtool folder and running the Update action. Afterwhich you can then delete those resources, if desired.

"How do I hide the status bar?"

This baserom comes with several objects to toggle features in levels, including the status bar. Check out the UberASM Objects documentation for details.

"Does this have Fast Retry?"

Yes! You can turn on the retry in a level using a custom object. The UberASM Objects has a section for the retry system with details.

"Why are my block graphics not looking right?"

Many of the custom block graphics in the baserom use ExGraphics, loading E17 into the FG2 slot in the Super GFX Bypass dialog will make the blocks appear as they are intended to in other levels.

"Is the baserom SA-1 compatible?"

Yes! The baserom's resources are all compatible with SA-1, but the baserom itself is configured to be FastROM by default. However to use SA-1 it only takes a couple steps to switch over to it:

  1. In the "project.toml" file in the buildtool folder, change the initial_patch setting to use the SA-1 option.
  2. Rebuild your project in Callisto to make the change.

"Why can't I run the initialization script?"

This is due to something in Windows called Execution Policies which is a security and safety feature to prevent the execution of malicious scripts, by preventing any script from being run at all (among other things).

You can however change the Execution Policy to something more permissive to allow you to run scripts on your computer. To do this launch PowerShell with the Run as administrator option, paste the following and hit Enter.

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned

Now you should be able to run the initialization script. When you are all finished you can set it back to the default for your computer:

Set-ExecutionPolicy -ExecutionPolicy Default
