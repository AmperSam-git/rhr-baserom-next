Frequently Asked Questions

"What is Callisto?"

It's a build system that makes it easier to build the rom from scratch, by doing this it avoids potential conflicts between resources and insures your hack is built cleanly.

"How do I remove things from the baserom?"

Callisto makes it easy to remove items from the baserom without a problem as it re-builds your ROM from scratch avoiding resource conflicts. Removing resources, like blocks or sprites, is as straightforward as editing the list of the respective tool and running Lunar Helper's "(Re)Build" action. You can then delete those resources, if desired.

"How do I hide the status bar?"

This baserom comes with several objects to toggle feature on/off in levels, including the Status bar. Check out the UberASM Objects documentation for details.

"Does this have Fast Retry?"

Yes! You can turn on the retry in a level using a custom object. The UberASM Objects has a section for the retry system with details.

"Why are my block graphics not looking right?"

The custom block graphics in the baserom use ExGraphics, loading E17 into the FG2 slot in the graphics override dialog will make the blocks appear as they are intended to in other levels.

"Is the baserom SA-1 compatible?"

Yes! The baserom's resources are all compatible with SA-1, but the baserom itself is configured to be FastROM by default. However to use SA-1 it only takes a couple steps to switch over to it:

  1. In the "project.toml" file in the Callisto folder, change the "initial_patch" setting to use the SA-1 option.
  2. Rebuild your project in Callisto to make the change
